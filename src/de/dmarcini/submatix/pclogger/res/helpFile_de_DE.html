Einführung

Dieses Programm verbindet sich mittels der Blutethooth Schnittstelle Ihres PC mit dem SPX42 Tauchcomputer. Es kann die Konfiguration auslesen und eine neue Konfiguration setzen.

Weiterhin kann das Logbuch des SPX42 Tauchcomputers ausgelesen und in einer lokalen Datenbank gesichert werden.

Optionen auf der Kommandozeile

Das Programm kann mit Kommandozeilenoptionen gestartet werden. Folgende Optionen sind möglich
--loglevel LEVEL
Der Parameter bestimmt, wie ausführlich das Programm seine Tätigkeit dokumentiert. Die Dokumentation des Programmes werden an die Konsole und in eine Logdatei ausgegeben.
Möglich sind folgende Parameter:
SEVERE (höchster Level) Nur Fehlermeldungen
WARNING Fehlermeldungen und Warnungen
INFO Info, Warnungen, Fehlermeldungen
CONFIG Konfigurationesmeldungen, alles hier drüber
FINE oder DEBUG DEBUG Meldungen (nur für Entwickler interessant)
FINER Noch ausführlicher
FINEST (niedrigster Level) Einfach alle Meldungen
--logfile path/file
Der Parameter bestimmt den Namen und die Position der Logdatei. Wird als Parameter NONE angegeben, wird keine Logdatei erzeugt.
Standartmäßig wird im Programmverzeichnis eine Datei mit dem Namen "logfile.log" angelegt.
Angabe mit absolutem Pfad, relativem Pfad oder ohne Pfad (Programmverzeichnis wird automatisch gewählt)
--cacheonstart
Der Parameter besitzt keine Werte. Wird dieser Parameter auf der Kommandozeile eingetragen, versucht die Bluethooth Bibliothek zunächst bereits evaluierte BT-Geräte ohne weitere Suche darzustellen.
Dies kann die Wartezeit erheblich verkürzen. Unter Max Os kann es zum "einfrieren" des Programmes führen.
