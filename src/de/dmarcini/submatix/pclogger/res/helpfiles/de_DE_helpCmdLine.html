Kommandozeilenoptionen

-logfile path/file
Der Parameter bestimmt den Namen und die Position der Logdatei. Wird als Parameter NONE angegeben, wird keine Logdatei erzeugt.
Standartmäßig wird im Programmverzeichnis eine Datei mit dem Namen "logfile.log" angelegt.
Angabe mit absolutem Pfad, relativem Pfad oder ohne Pfad (Programmverzeichnis wird automatisch gewählt)
--loglevel [SEVERE|ERROR|WARNING|INFO|CONFIG|FINE|DEBUG|FINER|FINEST]
Gibt an, welcher Detailgrad das programm beim Logging berücksichtigt
-cacheonstart
Der Parameter besitzt keine Werte. Wird dieser Parameter auf der Kommandozeile eingetragen, versucht die Bluethooth Bibliothek zunächst bereits evaluierte BT-Geräte ohne weitere Suche darzustellen.
Dies kann die Wartezeit erheblich verkürzen. Unter Max Os kann es zum "einfrieren" des Programmes führen.
-debug
Der Parameter besitzt keine Werte. Wird dieser Parameter auf der Kommandozeile eingetragen, werden alle Ausgwben des Programmes zusätzlich zur Ausgsabe in die Logdatei auch auf der Konsole ausgegeben. Nützlich bei Start mit der debugversion von java.
-databasedir
Mit diesem Parameter kann beim Start das Verzeichis angegeben werden, in dem die Datenbank gespeichert ist.
(Es ist im Normalfall nicht notwendig, diese Option zu benutzen)
-langcode
Normalerweise startet das Programm mit der Sprache, für welche der ausführende Computer eingerichtet ist. Wird diese Sprache (noch) nicht unterstützt, wird als Standart englisch benutzt.
Soll eine der unterstützten Sprachen, abweichend vom ausfühtrenden Computer benutzt werden, ist dieser Parameter hilfreich.
Soll beispielsweise französisch gewählt werden würde der Parameter "-langcode fr" lauten.
Der Landescode entspricht der Festlegung im "Two-Letter Code ISO-639"
