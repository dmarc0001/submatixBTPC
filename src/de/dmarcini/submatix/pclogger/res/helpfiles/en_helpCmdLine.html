Command Line Options

-logfile path/file
This parameter determines the logfile's name and position. If NONE is chosen as value no logfile is being creating.
Default setting creates a file 'logfile.log' inside the program's directory.
Declaration of path in absolute or relative path. Declaring no path will automatically select program directory
--loglevel [SEVERE|ERROR|WARNING|INFO|CONFIG|FINE|DEBUG|FINER|FINEST]
Determines the degree of detail the program uses for logging.
-cacheonstart
This parameter has no options. Using this command in the prompt will cause the program to search the bluetooth library for BT-devices already evaluated without displaying further searches.
This option causes sometimes an freeze under MAC OS.
-debug
This parameter has no options. Using this command in the prompt will cause the program to display all outputs not only in the logfile but also the terminal.
Useful when starting the debug version of java.
-databasedir
Using this parameter will cause the program to display the database's directory.
(Usually this is not necessary.)
-langcode
The program normally starts using the operating system's default language. If the program does not support this language, the standard English will be used.
The parameter enables the setting of a supported language other than the system's default.
If for example French is to be set, the parameter would be '-langcode fr'. The country code is determined by the 'Two-Letter Code ISO-639'.
